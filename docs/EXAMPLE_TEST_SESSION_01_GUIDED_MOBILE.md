# Fiche d'exemple de session de test 01 — Prototype mobile guide

## Statut du document

Cette fiche est une **session d'exemple remplie** a partir du protocole de test et de la documentation actuelle du prototype mobile. Elle montre comment utiliser `TEST_SESSION_TEMPLATE.md` dans un cas concret et credible. Elle ne doit pas etre lue comme une preuve terrain exhaustive.

## Identification

- Date : exemple de session a completer lors d'un vrai essai
- Lieu : demonstration interne ou preparation HKB, face a une reproduction ou a une discussion sur les oeuvres
- Personne qui observe / medie : etudiant·e ou mediateur·trice du projet
- Format teste : prototype mobile principal, mode `Parcours guide`
- Version ou fichier concerne : `web/index.html` + logique guidee documentee dans `docs/MOBILE_TESTING.md`
- Duree approximative : 5 a 7 minutes

## Question principale du test

- Que cherche-t-on a verifier dans cette session ?

Verifier si le parcours guide aide reellement a ralentir le regard, a clarifier l'entree dans les interactions et a reconduire le visiteur vers l'oeuvre plus nettement que l'exploration libre.

## Contexte

- Type de situation :
  - demonstration HKB
  - test entre pairs
- Nombre de personnes presentes :
  - 2 a 4
- Public concerne :
  - pair, enseignant, visiteur curieux, personne voyant le projet pour la premiere fois
- Conditions particulieres :
  - manque de temps
  - telephone personnel ou telephone de demonstration
  - mediation orale légère

## Profil general du visiteur ou du groupe

- Age approximatif ou type de public :
  - adulte ou pair HKB
- Familiarite avec art abstrait :
  - variable
- Familiarite avec smartphone / dispositif :
  - generalement bonne
- Commentaire utile :
  - la personne comprend vite l'usage du telephone, mais pas necessairement l'intention de mediation sans une phrase d'introduction

## Deroulement

- Comment l'entree dans l'experience a ete introduite :
  - une phrase courte du type : "Je vous propose une petite mediation de regard sur trois oeuvres, pas un jeu ni un questionnaire."
- Le visiteur a-t-il compris rapidement ce qu'il fallait faire :
  - partiellement ; l'opposition entre `Parcours guide` et `Explorer librement` est comprise, mais le sens du parcours guide doit encore etre verbalise
- Le dispositif a-t-il demande une explication orale supplementaire :
  - oui, legere, surtout au debut
- A quel moment principal une hesitation est apparue :
  - avant le premier geste, au moment de comprendre ce qui est attendu sans "faire trop"
- A quel moment principal une comprehension ou un engagement est apparu :
  - lorsque l'interaction devient visuellement lisible et que la logique du mot puis du retour a l'oeuvre s'enchaine

## Observation qualitative

### Comprehension

- comprehension immediate / tardive / fragile :
  - tardive mais solide apres le premier geste
- ce qui a ete compris :
  - la logique generale du parcours, le fait qu'il faut prendre son temps, la structure `observer -> geste -> mot -> retour`
- ce qui est reste flou :
  - le niveau exact d'action attendu avant le premier contact, et parfois la difference entre documentation et interaction pour `Mark looking`

### Rythme

- rythme trop rapide / juste / trop lent :
  - plutot juste si le mediateur n'intervient pas trop vite
- le visiteur a-t-il pris le temps :
  - oui, davantage en parcours guide qu'en exploration libre
- l'attention est-elle restee sur l'ecran, le materiau ou l'oeuvre :
  - elle passe par l'ecran, mais le parcours semble mieux preparer le retour a l'oeuvre que le mode libre

### Geste ou action

- le geste ou l'action etait-il lisible :
  - oui, surtout sur l'oeuvre 1 et sur les interactions les plus simples
- le changement produit etait-il perceptible :
  - oui, de maniere suffisante lorsque les invites visuelles sont bien remarquees
- le moment de passage a l'etape suivante etait-il clair :
  - globalement oui, grace au bouton `Choose a word`, mais ce moment doit rester tres sobre

### Retour a l'oeuvre

- le retour a l'oeuvre a-t-il eu lieu :
  - oui, dans la logique du protocole
- si oui, de quelle maniere :
  - au moment ou le visiteur est explicitement invite a baisser le telephone et a regarder encore la peinture
- si non, pourquoi probablement :
  - dans une situation de demonstration trop rapide, le regard risque de rester sur l'interface

## Verbatim ou mots emergents

- Mots choisis ou formules prononcees :
  - voile
  - trace
  - masse
  - silence
  - voisinage
- Citation courte eventuelle :
  - "Je comprends mieux quand l'image me montre moins et me fait attendre un peu."

## Traces produites

  - possible et recommande

  - possible si la session sert de test documente

  - utile en contexte HKB
- aucune :
  - non recommande si la session doit nourrir une comparaison

## Evaluation rapide

- Hesitation : moyenne
- Besoin de mediation orale : moyen
- Retour a l'oeuvre : moyen a fort
- Potentiel du format : fort

## Ce que nous avons appris

- apprentissage principal :
  - le parcours guide donne plus de coherence que le mode libre et porte mieux l'intention de mediation
- apprentissage secondaire :
  - la premiere entree dans le geste reste le point le plus sensible a clarifier

## Decision de suite

- garder mais simplifier
- clarifier la consigne

## Prochaine action recommandee

- prochaine petite modification ou prochain test :
  - tester le meme parcours avec une introduction orale encore plus courte pour voir ce qui reste compris sans aide
