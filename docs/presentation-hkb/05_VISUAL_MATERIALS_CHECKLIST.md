# Checklist des matériaux visuels

Cette checklist sert à préparer les images et supports nécessaires avant la soutenance HKB.

## 1. Visuels du prototype web

- [ ] capture de l'écran d'accueil montrant `Parcours guidé` et `Explorer librement`
- [ ] capture d'une interaction sur l'œuvre 1
- [ ] capture d'une interaction sur l'œuvre 2
- [ ] capture d'une interaction sur l'œuvre 3
- [ ] capture du moment de retour à l'œuvre
- [ ] capture du `Test summary`
- [ ] une vue du prototype sur écran mobile ou dans un mockup de téléphone

## 2. Accès et diffusion

- [ ] QR code vers la version GitHub Pages
- [ ] URL GitHub Pages vérifiée avant la présentation
- [ ] capture de secours si l'URL publique ne fonctionne pas
- [ ] capture ou note de la version locale `http://localhost:8000/web/`

## 3. Visuels Processing

- [ ] une ou deux captures du sketch Processing local
- [ ] une capture montrant une interaction ou une ambiance visuelle spécifique
- [ ] une capture utile pour faire sentir le lien possible avec projection ou installation

## 4. Schémas et scénarios

- [ ] schéma simple de la partition d'attention
- [ ] carte des prototypes du laboratoire
- [ ] diagramme comparant parcours guidé et exploration libre
- [ ] schéma de storyline ou de navigation préparé dans Figma ou dans un format équivalent
- [ ] diagramme d'une interaction, par exemple geste puis mot puis retour à l'œuvre

## 5. Pistes d'extension

- [ ] image ou schéma d'une hypothèse Arduino
- [ ] croquis d'une version projection ou installation
- [ ] visuel d'une version tablette ou mode kiosque
- [ ] petit schéma montrant comment ces pistes prolongent le laboratoire sans promettre une réalisation complète

## 6. Alternatives low-tech

- [ ] photo ou mockup de calques transparents
- [ ] photo ou mockup de cartes de couleur
- [ ] photo ou mockup de cartes de mots
- [ ] exemple de carte du regard sur papier
- [ ] visuel montrant une version orale ou accompagnée par médiateur

## 7. Visuels de documentation

- [ ] capture d'un résumé de test copiable
- [ ] capture d'un export JSON ou d'un extrait représentatif
- [ ] visuel expliquant l'absence de collecte de données personnelles
- [ ] slide simple sur les limites et les pistes abandonnées

## 8. Minimum à préparer si le temps manque

- [ ] écran d'accueil du prototype web
- [ ] une interaction forte sur l'œuvre 1
- [ ] QR code GitHub Pages
- [ ] carte des prototypes
- [ ] une capture du mode test ou du `Test summary`
- [ ] un visuel low-tech
- [ ] un schéma rapide d'une piste future
