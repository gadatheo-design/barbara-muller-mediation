# Carte des prototypes

## Statut

Vue d'ensemble synthétique du laboratoire. Cette carte sert à situer les différents prototypes, leurs relations et leurs degrés de maturité.

## Intention

Montrer que le projet ne se réduit pas à une seule interface mobile, mais qu'il s'organise comme un réseau de formats complémentaires.

## Description

Le laboratoire se compose aujourd'hui de plusieurs familles de prototypes :

| Piste | Fonction principale | Niveau actuel |
|---|---|---|
| Processing local | tester rapidement des interactions visuelles et des ambiances | prototype expérimental actif |
| p5.js web | rendre l'expérience testable sur smartphone et diffusable via le web | prototype public principal |
| Parcours guidé | structurer l'expérience en séquence de médiation | implémenté dans la logique du prototype web |
| Mode test / JSON | documenter les essais publics sans collecte invasive | opérationnel localement |
| Figma | formaliser des scénarios et des futurs écrans | outil de conception et de communication |
| Arduino | ouvrir une piste matérielle et sensorielle | hypothèse exploratoire |
| Projection / installation | déplacer l'expérience vers une forme collective | piste pour présentation HKB |
| Tablette | stabiliser un usage accompagné en groupe | piste intermédiaire crédible |
| QR code | faciliter l'accès dans le contexte d'exposition | modalité d'entrée simple |
| Low-tech | proposer une médiation sans dépendance technique | alternative complémentaire |

Cette carte n'est pas linéaire. Certaines pistes servent à produire, d'autres à tester, d'autres encore à communiquer le projet ou à en imaginer la suite.

## Expérience visiteur

Pour le visiteur, cette pluralité peut se traduire par plusieurs portes d'entrée : une expérience individuelle sur téléphone, une médiation conduite par une personne, une interaction collective projetée, ou une activité papier plus légère. Le laboratoire cherche précisément à comparer ces situations.

## Rôle du médiateur

Le médiateur aide à choisir la bonne porte d'entrée. Il peut activer une version simple pour une rencontre brève, un parcours guidé pour un accompagnement plus attentif, ou une piste low-tech si le contexte technique ne permet pas l'usage d'un écran.

## Ce que nous avons appris

La cartographie des prototypes rend visible un point important : le projet progresse par bifurcations et reformulations. Ce déplacement a clarifié la valeur du travail. La question n'est pas seulement "quelle application construire ?" mais aussi "quelle forme de médiation produit quel type d'attention ?".

## Limites actuelles

La carte reste un outil de lecture. Elle ne remplace ni des tests plus nombreux ni une évaluation comparative approfondie entre les formats.

## Développements possibles

Cette carte pourrait devenir un support de soutenance, un schéma mural, ou un document partagé avec des enseignants et des médiateurs pour discuter des choix à poursuivre après le cours.
